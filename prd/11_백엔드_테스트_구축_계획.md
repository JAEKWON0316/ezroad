# 백엔드 테스트 인프라 구축 계획

## 목표 설명
Spring Boot 백엔드를 위한 견고한 테스트 환경을 구축합니다. 필요한 의존성을 설정하고, 테스트 디렉토리 구조를 잡은 뒤, 설정을 검증하기 위한 초기 단위 및 통합 테스트를 작성합니다.

## 사용자 검토 필요 사항
> [!IMPORTANT]
> `spring-boot-starter-test`와 독립적인 DB 테스트를 위한 `h2` 데이터베이스 의존성을 추가할 예정입니다.
> `backend/src/test` 디렉토리를 새로 생성합니다.

## 변경 제안

### 백엔드 설정
#### [수정] [build.gradle](file:///c:/linkisy/ezenroad/backend/build.gradle)
- `testImplementation 'org.springframework.boot:spring-boot-starter-test'` 추가
- `testImplementation 'org.springframework.security:spring-security-test'` 추가
- `testRuntimeOnly 'com.h2database:h2'` 추가 (독립적인 DB 테스트용)

### 테스트 구조 설정
#### [신규] [backend/src/test/java/com/ezroad/EzRoadApplicationTests.java](file:///c:/linkisy/ezenroad/backend/src/test/java/com/ezroad/EzRoadApplicationTests.java)
- Spring Boot 컨텍스트 로드 테스트 (기본 설정 검증)

#### [신규] [backend/src/test/java/com/ezroad/service/MemberServiceTest.java](file:///c:/linkisy/ezenroad/backend/src/test/java/com/ezroad/service/MemberServiceTest.java)
- Mockito를 사용한 `MemberService` 단위 테스트
- DB 의존성 없이 비즈니스 로직 검증

#### [신규] [backend/src/test/java/com/ezroad/controller/AuthControllerTest.java](file:///c:/linkisy/ezenroad/backend/src/test/java/com/ezroad/controller/AuthControllerTest.java)
- `MockMvc`를 사용한 인증 API 통합 테스트
- HTTP 요청/응답 처리 및 Security 설정 검증

## 검증 계획
