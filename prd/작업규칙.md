## 🔧 MCP 사용 규칙

### MCP 사용 전 반드시 확인
모든 MCP 도구 사용 전에 사용자에게 먼저 물어볼 것:
- "이 작업에 [MCP 이름]을 사용하려고 합니다. 진행할까요?"
- 예: "Neon DB 테이블을 생성하려면 Supabase MCP를 사용해야 합니다. 진행할까요?"

작업 중 MCP가 필요할것 같으면 최대한 활용할것
### 사용 가능한 MCP 목록
1. **Supabase MCP** - Neon DB 연결 시 (참고: Neon은 별도 MCP 없음)
2. **Vercel MCP** - 프론트엔드 배포
3. **Desktop Commander** - 로컬 파일 작업
4. **Playwright MCP** - 브라우저 테스트

## 📝 문서 작성 규칙

### 파일명 규칙
- 숫자 prefix로 순서 표시: `01_`, `02_`, ...
- 한글 사용 가능
- 예: `01_Phase1_인프라설정.md`

### 문서 구조
1. 제목 (H1)
2. 목표/개요
3. 상세 내용
4. 체크리스트 (작업 항목)
5. 관련 문서 링크

### 상태 표시
- ⬜ 대기
- 🔄 진행중
- ✅ 완료
- ❌ 보류

## 💻 코드 작성 규칙

### Frontend (Next.js)
- TypeScript 필수
- 컴포넌트: PascalCase (`RestaurantCard.tsx`)
- 유틸/훅: camelCase (`useAuth.ts`)
- App Router 사용

### Backend (Spring Boot)
- Java 17+
- 패키지: `com.ezenroad`
- Entity → Repository → Service → Controller 순서
- DTO는 request/response 분리

### 커밋 메시지
```
[타입] 간단한 설명

- feat: 새 기능
- fix: 버그 수정
- docs: 문서
- refactor: 리팩토링
- style: 포맷팅
```
