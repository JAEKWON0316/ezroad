# 백엔드 테스트 인프라 구축 완료 보고서

## 개요
Spring Boot 백엔드 프로젝트의 안정성을 확보하기 위해 테스트 인프라를 구축했습니다. JUnit 5, Mockito, MockMvc를 활용하여 단위 테스트와 통합 테스트 환경을 구성했습니다.

## 변경 사항

### 1. 의존성 추가 ([build.gradle](file:///c:/linkisy/ezenroad/backend/build.gradle))
- `spring-boot-starter-test`: JUnit, Mockito 등 기본 테스트 라이브러리
- `spring-security-test`: Spring Security 관련 테스트 지원
- `h2`: 테스트용 인메모리 데이터베이스

### 2. 테스트 코드 작성

#### 기본 컨텍스트 로드 테스트 ([EzRoadApplicationTests.java](file:///c:/linkisy/ezenroad/backend/src/test/java/com/ezroad/EzRoadApplicationTests.java))
- Spring Boot 애플리케이션 컨텍스트가 정상적으로 로드되는지 검증

#### MemberService 단위 테스트 ([MemberServiceTest.java](file:///c:/linkisy/ezenroad/backend/src/test/java/com/ezroad/service/MemberServiceTest.java))
- `Mockito`를 사용하여 Repository와 외부 의존성을 Mocking
- 회원가입 비즈니스 로직 검증 (비밀번호 암호화, 중복 체크 등)
- **주요 해결 사항**: `Member` 엔티티의 Builder 패턴 이슈를 해결하기 위해 `ReflectionTestUtils`를 사용하여 ID 주입

#### AuthController 통합 테스트 ([AuthControllerTest.java](file:///c:/linkisy/ezenroad/backend/src/test/java/com/ezroad/controller/AuthControllerTest.java))
- `MockMvc`를 사용하여 실제 HTTP 요청/응답 흐름 테스트
- 회원가입 및 로그인 API 엔드포인트 검증
- JSON 요청/응답 직렬화 및 역직렬화 확인

## 테스트 실행 방법

터미널에서 다음 명령어를 실행하여 모든 테스트를 수행할 수 있습니다.

```bash
cd backend
./gradlew test
```
